The ACE/Agent Administration Applet

You must be a Netscape Server administrator with "Write" permission on the netscape_home_directory/plugins/aceagent directory to administer the ACE/Agent settings.

In addition, your username (for example, doej or pmarlowe) must be listed in the AdminList parameter of the aceagent.cfg file.

Multiple ACE/Agent administration sessions are not supported. If two or more administrators attempt to manipulate WebID settings at the same time, one administrator could possibly undo settings that the other administrator(s) has made.

To run the ACE/Agent Administration applet:

  1. Start your Web browser.
       
  2. Point your browser to http [or https]://servername.your_domain/aceagent/index.html (open admin.html for a "no frames" version).
     Note: The ACE/Agent Administration applet is accessed through the public Web server port, not the server's administration port.
       
  3. Click the Administer ACE/Agent button in the top left-hand frame.
         
  4. The ACE/Agent Administration applet will appear in a secondary browser window, allowing you to browse the Help topics in the primary window while running the applet.

The ACE/Agent Administration applet controls the following ACE/Agent configuration settings:

Designating which resources to protect

Distributing WebID cookies

Configuring cookie expiration controls

Distributing domain cookies

Ensuring that every connection is secure

Preventing users from caching protected pages

Remote Administration

To use the ACE/Agent Administration applet from a remote machine, you must authenticate using your SecurID token to obtain a valid WebID cookie. If you have expiration controls enabled on your cookies, you will be prompted to reauthenticate when the cookie expires.

Security Dynamics also recommends that you perform all remote ACE/Agent administration over a secure connection (https).
