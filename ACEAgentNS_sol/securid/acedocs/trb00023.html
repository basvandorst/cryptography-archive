Viewing the Contents of sdconf.rec

The ACE/Agent software uses the sdconf.rec file to locate the ACE/Server and to configure authentication parameters unique to your ACE/Server environment (for example, required PIN length, number of bad PASSCODEs allowed, etc.). After ACE/Agent installation, you should have obtained a copy of the sdconf.rec file from your ACE/Server administrator and specified its location in the VAR_ACE parameter of the netscape_home_directory/plugins/aceagent/aceagent.cfg file.

The sdconf.rec file may become corrupted or may contain incorrect information that could inhibit proper user authentication. For this reason, it is helpful to view the contents of the sdconf.rec file as a troubleshooting measure. If your sdconf.rec file has been corrupted or contains incorrect or outdated information, you must obtain a new copy of the file from your ACE/Server administrator.

To view the contents of the sdconf.rec file:

  1. Set the VAR_ACE environment variable to the sdconf.rec location that you specified during ACE/Agent installation.
        
  2. Set the USR_ACE environment variable to the netscape_home_directory/plugins/aceagent directory.
        
  3. cd to the netscape_home_directory/plugins/aceagent directory.
        
  4. Run the sdinfo program.
