PGP 공개 전자우편 키서버 (PGP Public Email Keyservers)

PGP 공개 전자우편 키서버는 인터넷과 UUCP 우편시스템을 이용하여 서로의 공개키(public keys)를 교환해 주고 있습니다.

WWW 를 이용할 수 있으면 보다 편리한 WWW 환경을 사용할 수 있으며 자주 공개키를 검색할 필요가 있는 분들은 FTP 서버 에서 모든 공개키가 들어있는 전체 키링(keyring) 파일을 받아 가실 수 있습니다.

이 서비스는 PGP 사용자들이 공개키를 교환하는데 도움을 줄 뿐 사용자들 이 등록한 공개키의 등록자에 대한 진위여부는 책임지지 않습니다. 그러한 종류의 보안을 원한다면 서명을 붙인 키를 사용하십시오.

모든 키서버는 다음과 같은 형식의 전자우편을 처리합니다. 키서버가 처리할 명령은 항상 제목(Subject): 줄에 들어가야 하므로 전자우편의 본 내용(body) 부분에 들어가지 않도록 주의해 주십시오. 

        To: pgp-public-keys@keys.kr.pgp.net
        From: kdhong@some.site.edu
        Subject: help
여러분의 공개키는 하나의 키서버에만 등록하면 다른 키서버에도 자동으로 등록됩니다.

예를 들어 여러분이 자신의 공개키를 키서버에 등록하거나 이미 등록된 공 개키를 바꾸고자 한다면 아래와 같은 형식의 전자우편을 아무 키서버에나 보내십시오. 

        To: pgp-public-keys@keys.kr.pgp.net
        From: kdhong@some.site.edu
        Subject: add

        -----BEGIN PGP PUBLIC KEY BLOCK-----
        Version: 2.6.3i

        <등록할 공개키가 여기 들어갑니다>
        -----END PGP PUBLIC KEY BLOCK-----
폐기된 키들: 먼저 Key Revocation Certificate 를 만들고 (만드는 방법은 PGP 설명서에 나와 있슴) 키서버에 ADD 명령으로 키를 보내십시오.

키서버가 처리할 수 있는 명령들: 

명령                   설명


ADD                    공개키를 등록 (등록할 공개키는 전자우편의 본내용(body) 부분에 들어가야 함)
INDEX                  등록된 PGP 공개키들의 목록을 보여 줌 (-kv)
INDEX userid           등록된 PGP 공개키들 중 'userid' 가 들어 있는 키들의 목록을 보여 줌 (-kv)
VERBOSE INDEX          등록된 PGP 공개키들의 목록을 verbose 형식으로  보여 줌 (-kvv)
VERBOSE INDEX userid   등록된 PGP 공개키들 중 'userid' 가 들어 있는 키들의 목록을 verbose 형식으로 보여 줌 (-kvv)
GET                    (주의!) 등록된 PGP 공개키를 모두 보여 줌 (split)
GET userid             'userid' 의 PGP 공개키를 보여 줌
MGET regexp            '/regexp/' 에 맞는 PGP 공개키를 모두 보여 줌 (이때 'regexp'는 적어도 2 글자(charater) 이상이어야 함)
LAST days              최근 'days' 일 동안 변경된 PGP 공개키를 모두 보여 줌
MGET 명령을 사용한 예: 
        MGET kdhong             이름에 "kdhong"이 포함된 모든 키를 보여 줌
        MGET seoul              이름에 "seoul"이 포함된 모든 키를 보여 줌
        MGET E8F605A5|5F3E38F5  주어진 두개의 키번호(keyid)와 일치하는 모든 키를 보여 줌
regexps에 관하여: '*'는 Unix 쉘(shell)이나 MSDOS 에서 사용하던 것과 달리 '바로 전의 글자(character)와 일치하는 여러개의 글자들 또는 공백' 을 의미합니다. 
	a.*  는 a 로 시작하는 모든 단어를 의미합니다
	ab*c 는 ac, abc, abbc 등의 단어를 의미합니다.
(주의!) MGET .* 명령을 사용하는 대신 GET 명령을 사용하십시오.

GET 명령에 관하여: 가능한 한 GET 명령을 사용하는 대신 FTP 서버를 통해 전체 키링(keyring) 파일을 가져 가십시오. 현재 키링 파일은 각각이 52K 인 파일로 50개가 넘으므로 GET 명령의 사용은 많은 부담을 주게 됩니다.

전자우편 키서버 사용자들은 전자우편을 보낼 때 pgp-public-keys@keys.pgp.neta 주소를 이용하거나 아래의 주소중 사용자의 국가에 있는 전자우편 키서버 주소를 이용하십시오:

    pgp-public-keys@keys.uk.pgp.net
    pgp-public-keys@keys.de.pgp.net
    pgp-public-keys@keys.no.pgp.net
    pgp-public-keys@keys.us.pgp.net
    pgp-public-keys@keys.nl.pgp.net
    pgp-public-keys@keys.fi.pgp.net
    pgp-public-keys@keys.es.pgp.net
    pgp-public-keys@keys.hr.pgp.net
FTP 서비스를 원하시면 ftp.pgp.net:pub/pgp/으로 접속해 주십시오.

위에 나열된 '*.pgp.net' 주소들은 안정되고 확실하므로 이 주소들을 사용하시기 바랍니다.
