Startup Message: sdconf.rec not found

Meaning: The ACE/Agent could not locate the sdconf.rec file.

Solution: Check the var_ace directory specified in aceagent.cfg for the sdconf.rec file. If you have not obtained a copy of this file, contact your ACE/Server administrator.

Note: If the VAR_ACE environment variable is set at the command line during server startup, the VAR_ACE directory specified in aceagent.cfg will be overridden by the value of the VAR_ACE environment variable.
