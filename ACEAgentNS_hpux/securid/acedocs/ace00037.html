Protecting a Specific Directory on the Server

To enable Web protection on a specific directory:

Directory protection is preferred because any files or subdirectories you add to the directory later will be protected automatically.

  1. Start the ACE/Agent Administration applet.
     
  2. Click Select files to secure. The Select Files to Secure dialog box opens. The Select Files to Secure dialog box is laid out as a standard directory tree, with the folder icon at the top of the tree indicating the root or “home” directory of the server. You can browse the contents of the server by single-clicking the + and - boxes next to the folder icons.
     
  3. Browse through the tree to find and select the directory you want to protect. For example, if in the URL http://www.home.com/products/ you wanted to protect the contents of products, you would select the products directory. (To select a series of directories, hold down the SHIFT or CNTL key and click the multiple directory names.)
     
  4. Click the Secure button. All the subordinate directories in the tree disappear and a SecurID icon appears in place of the folder icon, indicating that its entire contents are SecurID protected.
     
  5. Click OK to confirm the changes.

Note: Any files or subdirectories that you add to the directory in the future will automatically be SecurID protected.
