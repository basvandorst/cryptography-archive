Differences between JavaSoft's version of JCE
and the International JCE

This documentation is not completed - it will go into much more detail about each of the following:

  • getAlgorithm methods.
  • Ciphers with differing plaintext and ciphertext block sizes.
  • Cipher.update methods.
  • Additional Cipher constructors.
  • Static creation of ciphers.
  • Differences in security checking for the Security class (Java 1.0.2 only).
  • Security fix to Provider class.
  • implPadding and implBuffering.
  • Mode classes.
  • PaddingScheme vs. Padding.
  • Support for padding of arbitrary block sizes.
  • Selection of PaddingScheme and Mode implementations, when not available from the requested provider.
  • java.security.IJCE class.
  • In IJCE, java.security.interfaces.RSAPrivateKey extends PrivateKey, and java.security.interfaces.RSAPublicKey extends PublicKey (in JavaSoft's version they do not, which was probably an unintentional omission).
  • java.security.interfaces.RSAFactors allows RSA operations to be optimized using the Chinese Remainder Theorem, even when using key objects from a different provider.
