Cryptix TM Compilation

To compile, see the scripts in the util directory and also the src/build.* scripts.

Here are some examples.

  • This will create a binary distribution in the build directory:

    • Under Unix, enter 
              sh util/finalbuild.sh

    • Under NT or Windows 95 (from a command prompt), enter 
              util\finalbuild
    To test the above, ensure that the classes archive that was built, build/cryptix.zip, is in your CLASSPATH.

  • To compile the classfiles to be in the same directories as the source, you should use the build script instead. 
        cd src
    build                     # Unix: sh build.sh

  • The scripts that are used to create a binary distribution are provided. To build the IJCE into a JAR (Unix only): 
        util/buildjar.sh -verbose util/man/ijce.sh
    and then, to build the Cryptix provider: 
        util/buildjar.sh util/man/spt.man
    util/buildjar.sh util/man/alg.man

    By default, JARs are put in dist.o directory. These can then be tested with testjar.sh: 

        cd src/dist.o
    ../../util/testjar.sh IJCE_*.jar
    ../../util/testjar.sh SPT_*.jar
    ../../util/testjar.sh ALG_*.jar

    Note that there are currently no tests within the IJCE JAR (it is tested by TestIJCE within the ALG JAR).

  • To build a full binary distribution, this command is used: 
        util/dist_binary.sh util/man/binary.man
    which creates all the JARs as instructed in the manifest and packages them into a zip. It doesn't succeed unless all the tests succeed.
